Cyber Threats to be aware of and hardening recommendations to secure your website:

  • Cross-Site Scripting (XSS) Prevention:

    • When it comes to developing websites there are attacks that can occur known as XSS which is a malicious script that is injected into a website, it gives attackers privlege to steal user data or hijack sessions. Website developers have the obligation to prevent XSS attacks by sanitizing user inputs, using Content Security Policy (CSP) headers, and escaping output data. There are certain tools that can ehance the cyber protection of websites such as OWASP's XSS Filter which is a Open Web Application Secuirty Project. This tool is very useful approach to provide security and as a non-profit organization OWASP's gives customers the resources and guidance to safeguard against XSS attacks

  • SQL Injection Prevention:

    • When it comes to the severity of attachs that a website can experience, an SQL injection is one of the most severe web application vulnerability where attackers manipulate SQL queries by injecting malicious input. The consequences of this attack can lead to unauthorized access, data breaches, and full control of an website. This vulnerability shows the importance of conducting prevention methods through the use of parameterized queries. The practice of parametrized queries esnsure that users are able to input that serves as data as opposed to executable SQL code. Another tactic to take when it comes to securing an website would be the implementation of an effective web application firewall. The Web Application Firewall is a substantial tool that detects and blocks SQL injection attempts. Some of the most well known WAF comes from ModSecurity, Cloudflare WAF, and AWS WAF.

  • Cross-Site Request Forgery (CSRF):

    • This vulnerablity is a common web security vulnerability that gives an attacker the potential of deceiving a user into performing unintended actions on a trusted website without their authentic consent. These attacks can lead to major consequences that impact business transactions, data modification, and account changes. There are certain prevention methods to take when attempting to prevent CSRF attacks on your website. The ability of using anti-CSRF tokens will greatly enhance the security of a website through giving attackers a random token that is unique per session, and expires after each use. The CSRF vulnerability is a very serious weakness in many websites, so it is imperative to make sure you have the proper defense mechanisms that can prevent the attack from occuring.
