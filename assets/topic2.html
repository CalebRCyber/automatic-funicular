Introduction to Topic 2: Firewalls in Web Application Security

The implementation of firewalls is a proven cyber security measure that must be taken for any organization to practice within the confides of the CIA triad which applies to web development. This is a critical component of web appplicaiton securty due to the purpose of providing a barrier between trusted internal networks and untrusted external traffic that comes through the internet. The benefits of firewalls surpass the protocols of blocking malicious traffic that prevents security threats from incidents such as data breaches. This protocol reduces the possibility of SQL injection, cross-site scripting, denial of service attacks, and unauthorized access attempts.

Key Features of Web Development firewalls:

  • Network Firewalls

    • This crucial protcol serves as a security system that monitors incoming and outgoing network traffic that is based on established security protocols. When it comes to the nature of web development this stage of security is often the first line of defense that prevents unathorized access. The main categories of network firewalls is the protocols of packet filtering, stateful inspection, access control lists, and intrusion detection.

  • Web Application Firewalls (WAFs) in Web Development:

    • This security tool is essential for the protection of web applications and APIs from critical threats that come through online vulnerabilites. The WAF operates as the application layer of the OSI model that examines HTTP and HTTPS traffic by detecting suspicious activity. The main operation of WAF is to promote traffic inspection and threat detection that every web site application should implement.

  • Common Threats of Web Application development:

    • There are numerous cyber threats to be aware in the process and implementation of web applications. One of the most common threats relates to the input validation and sanitization that refers to the user input category such as form fields, URL parameters, and API requests. The threat of SQL injection, Cross-Site Scripting, and command injection attacks is a direct result of poor input validation and sanitization.
